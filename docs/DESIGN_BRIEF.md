# Design Brief

## What We Are Building

A playable web game with AI participation.

The first version should feel like a small text adventure / MUD-inspired mystery, not like an open-ended AI chat.

## What Makes It Fun

- clear goal
- limited time
- rooms to explore
- NPCs with trust
- clues to discover
- choices with costs
- visible consequences
- multiple endings

## What AI Adds

- better narration
- reactive NPC dialogue
- memory summaries
- consistency review
- clue presentation

## What AI Must Not Replace

- rules
- inventory
- room graph
- quest state
- action validity
- ending conditions

## First Acceptance Test

A player can open the web page, play for 5-10 minutes, make meaningful choices, and reach an ending without needing to understand the architecture.
